# UI 走查 Prompt

## 用途
系统化检查界面设计问题，确保视觉一致性和用户体验。

## 适用场景
- 设计稿评审
- 开发完成后的视觉验收
- 竞品分析

---

## Prompt 内容

```
请对以下界面进行 UI/UX 走查，指出设计问题和改进建议。

界面描述/截图：[描述界面或上传截图]
平台：[Web/iOS/Android/小程序]
设计规范：[是否有设计系统/组件库，如有请说明]

请按以下维度检查：

## 1. 视觉层
- **布局**：对齐是否一致、间距是否规范、视觉层级是否清晰
- **色彩**：是否符合品牌色、对比度是否足够（WCAG）、是否有过度用色
- **字体**：字号层级是否合理、行高是否舒适、字重使用是否一致
- **图标**：风格是否统一、尺寸是否规范、语义是否清晰

## 2. 交互层
- **状态**：是否考虑了空状态、加载态、错误态、极限数据态
- **反馈**：操作后是否有明确反馈、动效是否自然
- **手势/操作**：点击区域是否足够、是否支持常见手势

## 3. 信息层
- **文案**：是否简洁易懂、语气是否一致、是否有歧义
- **信息层级**：重要信息是否突出、次要信息是否弱化得当
- **空值处理**：无数据时如何展示

## 4. 一致性
- 与产品其他页面是否风格一致
- 与平台规范（iOS HIG / Material Design）是否冲突

---

输出格式：

| 问题位置 | 问题描述 | 严重程度 | 修改建议 |
|----------|----------|----------|----------|
| [具体位置] | [描述问题] | 高/中/低 | [具体建议] |

最后给出优先级最高的 3 个问题和总体评价。
```

---

## 使用技巧

如果是设计稿评审，可以加上：
```
请从开发可实现性角度评估，指出哪些效果可能有技术风险。
```
