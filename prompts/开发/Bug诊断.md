# Bug 诊断 Prompt

## 用途
帮助系统性分析和定位 Bug，不盲目猜测根因。

## 适用场景
- 遇到难以定位的 Bug
- 线上故障排查
- 学习如何结构化分析问题

---

## Prompt 内容

```
我遇到了一个 Bug，请帮我系统性地分析和定位。

以下是已知信息：
- 现象：[描述你看到的现象]
- 复现步骤：[如何复现]
- 预期行为：[应该发生什么]
- 实际行为：[实际发生了什么]
- 环境信息：[操作系统、浏览器、版本等]
- 相关代码：[粘贴相关代码]
- 已尝试的排查：[你试过哪些方法]
- 错误日志：[如果有，粘贴日志]

请按以下步骤帮我分析：
1. 先确认信息是否足够，不足的话向我提问
2. 列出所有可能的根因假设（不局限于代码本身）
3. 针对每个假设，给出验证方法（从最简单到最复杂）
4. 根据概率排序，建议我按什么顺序排查
5. 给出预防此类 Bug 再次发生的建议

不要直接给出"答案"，而是教我如何定位问题。
```

---

## 使用技巧

信息越完整，AI 给出的假设越准确。至少提供：现象、复现步骤、相关代码、错误日志。
