# 代码审查 Prompt

## 用途
系统化审查代码，发现潜在问题并提供可执行的改进建议。

## 适用场景
- 提交 PR 前的自查
- 团队代码审查
- 学习他人代码时理解设计决策

---

## Prompt 内容

```
请对以下代码进行全面的代码审查，按以下维度检查：

1. **正确性**：是否有逻辑错误、边界情况未处理、并发问题
2. **可读性**：命名是否清晰、函数是否过长、是否有必要的注释
3. **可维护性**：是否违反单一职责、耦合度如何、是否易于测试
4. **性能**：是否有明显的性能瓶颈、不必要的计算或内存泄漏
5. **安全性**：是否有注入风险、XSS、敏感信息泄露等问题
6. **一致性**：是否遵循项目已有的代码风格和模式

对于每个发现的问题：
- 指出具体位置（行号）
- 说明问题类型和严重程度
- 给出修改建议（可直接使用的代码片段）

最后给出一个总体评价（优秀/良好/需要改进）和最关键的 3 个改进建议。
```
