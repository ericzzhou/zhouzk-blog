# 文章大纲生成 Prompt

## 用途
根据主题快速生成结构清晰的文章大纲，适用于技术博客、公众号长文等。

## 适用场景
- 开始写作前的框架梳理
- 确保文章逻辑完整
- 快速评估文章工作量

---

## Prompt 内容

```
请帮我为以下主题生成一篇文章大纲。

主题：[文章主题]
目标读者：[读者画像，如：初级前端工程师]
文章类型：[技术教程/经验总结/观点论述/案例分析]
预期字数：[如：3000字]
核心观点：[你想传达的最重要的1-2个观点]

请按以下结构输出大纲：

## 建议标题（3个选项）
1. [标题A]
2. [标题B]
3. [标题C]

## 文章大纲

### 引言（占全文 10%）
- 钩子：[用什么吸引读者继续读]
- 背景：[为什么要关心这个话题]
- 预告：[本文会讲什么]

### 正文第一部分：[小标题]
- 要点1
- 要点2
- 案例/数据支撑

### 正文第二部分：[小标题]
...

### 正文第三部分：[小标题]
...

### 结尾（占全文 10%）
- 总结核心观点
- 行动号召或开放性问题
- 延伸阅读推荐

## 写作建议
- [针对这个主题的具体建议，如：注意什么坑、需要调研什么]
- [可选的素材建议，如：可以引用什么报告/案例]

---

要求：
1. 逻辑递进：从已知到未知，从问题到解决方案
2. 每个部分都有明确的价值点
3. 小标题要具体，不能是"第一部分"这种废话
4. 控制深度：适合[目标读者]的水平
```

---

## 进阶用法

如果已有一些素材，可以加上：
```
我已有以下素材：
- [素材1]
- [素材2]
请确保大纲能容纳这些内容。
```
